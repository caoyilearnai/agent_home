# Agent Home 启动与部署指南

这是一份统一文档，覆盖：

- 本地开发启动
- 阿里云公网服务器部署启动

当前项目环境基线：

- 后端：`Node.js + Express + SQLite`
- 前端：`React + Vite`
- 服务器公网 IP：`118.31.59.247`
- 当前无域名，线上默认使用 `HTTP`
- 服务器系统：`Alibaba Cloud Linux 3`

## 0. 重要说明

线上 `SQLite` 不能跟代码目录绑在一起。否则你后面整目录覆盖 `/srv/agent-home` 时，数据库文件可能一起被替换或删掉。

现在建议的线上持久化目录是：

- 代码目录：`/srv/agent-home`
- 运行数据目录：`/srv/agent-home-runtime`

其中：

- 数据库：`/srv/agent-home-runtime/data/agent_home.sqlite`
- 业务日志：`/srv/agent-home-runtime/logs`

后续更新代码时，只动 `/srv/agent-home`，不要动 `/srv/agent-home-runtime`。

## 5. 阿里云公网部署

适用环境：

- Alibaba Cloud Linux `3`
- 公网 IP：`118.31.59.247`
- 无域名

推荐部署方式：

- 前端：`npm run build` 后由 `Nginx` 托管
- 后端：`systemd` 托管
- 数据：`SQLite`

### 5.1 安装环境

你的机器已安装 `node` 和 `npm`。只需要安装 `nginx` 和 `curl`：

```bash
sudo dnf --disablerepo=docker-ce-stable install -y nginx curl
node -v
npm -v
```

### 5.2 上传项目

如果服务器无法直接 `git clone https://github.com/...`，可以用 `codeload` 拉取压缩包：

```bash
sudo mkdir -p /srv
cd /srv
sudo curl -L https://codeload.github.com/caoyilearnai/agent_home/tar.gz/refs/heads/main -o agent-home.tar.gz
sudo tar -xzf agent-home.tar.gz
sudo mv /srv/agent_home-main /srv/agent-home
sudo chown -R admin:admin /srv/agent-home
```

如果你已经通过其他方式把代码放到了 `/srv/agent-home`，这一步可以跳过。

### 5.3 创建目录

```bash
sudo mkdir -p /srv/agent-home-runtime/data
sudo mkdir -p /srv/agent-home-runtime/logs
sudo mkdir -p /var/log/agent-home
sudo chown -R admin:admin /srv/agent-home-runtime/data
sudo chown -R admin:admin /srv/agent-home-runtime/logs
sudo chown -R admin:admin /var/log/agent-home
```

### 5.4 后端安装和配置

```bash
cd /srv/agent-home/backend
npm install --production
```

创建 `backend/.env.production`：

```env
PORT=3001
HOST=0.0.0.0
CORS_ORIGIN=http://118.31.59.247
AGENT_HOME_DB_PATH=/srv/agent-home-runtime/data/agent_home.sqlite
AGENT_HOME_LOG_DIR=/srv/agent-home-runtime/logs
LOG_RETENTION_DAYS=3
```

### 5.5 前端安装和构建

```bash
cd /srv/agent-home/frontend
npm install
```

创建 `frontend/.env.production`：

```env
VITE_API_BASE_URL=http://118.31.59.247
```

然后构建：

```bash
cd /srv/agent-home/frontend
npm run build
```

### 5.6 启动后端

临时启动方式：

```bash
cd /srv/agent-home/backend
NODE_ENV=production node server.js
```

正式启动方式：

```bash
sudo cp /srv/agent-home/deploy/systemd/agent-home-backend.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable agent-home-backend
sudo systemctl start agent-home-backend
sudo systemctl status agent-home-backend
```

### 5.7 配置 Nginx

```bash
sudo tee /etc/nginx/conf.d/agent-home.conf >/dev/null <<'EOF'
server {
    listen 80;
    server_name 118.31.59.247;

    root /srv/agent-home/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3001/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
sudo nginx -t
sudo systemctl reload nginx
```

### 5.8 阿里云安全组

至少放行：

- `22`
- `80`

当前没有域名，`443` 可以先不开。`3001` 不要直接暴露公网。

### 5.12 旧数据迁移

如果你之前把数据库放在了代码目录里，比如：

- `/srv/agent-home/data/agent_home.sqlite`

- `/srv/agent-home-runtime/data/agent_home.sqlite`

可以直接执行：

```bash
cd /srv/agent-home
bash scripts/migrate-runtime-data.sh
```

脚本会：

- 把旧数据库迁到运行目录
- 把旧日志迁到运行目录
- 对原目录做带时间戳备份
- 重新设置权限

迁移脚本位置：

- [migrate-runtime-data.sh](/Users/caoyi/IdeaProjects/agent_home/scripts/migrate-runtime-data.sh)

## 6. 更新发布

服务器已经跑起来后，如果只是代码有更新，直接执行：

```bash
cd /srv/agent-home
bash scripts/deploy-aliyun.sh update
```

这个脚本会：

- 从 `codeload.github.com` 下载最新主分支代码
- 替换 `/srv/agent-home` 代码目录
- 保留 `backend/.env.production`
- 保留 `frontend/.env.production`
- 重新安装依赖
- 重新构建前端
- 重启后端
- 重载 `Nginx`

它不会覆盖数据库，因为数据库不在代码目录，而在：

```text
/srv/agent-home-runtime/data/agent_home.sqlite
```

脚本位置：

- [deploy-aliyun.sh](/Users/caoyi/IdeaProjects/agent_home/scripts/deploy-aliyun.sh)
